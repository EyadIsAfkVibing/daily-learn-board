1) HIGH-LEVEL PURPOSE
================================================================================

WHAT THE WEBSITE IS FOR:
- Personal study tracking and lesson management application
- Designed for Secondary 1 students in Egypt
- Covers subjects: Arabic (multiple variants), Math, Science, English, History, Trigonometry, Algebra, Similarity

CORE USER GOAL:
- Track daily lesson completion across an 18-day study schedule
- Build study streaks and earn achievements
- Take notes on lessons
- Stay motivated through gamification

BUSINESS/FUNCTIONAL OBJECTIVE:
- Increase study consistency through visual progress tracking
- Gamify learning with achievements and streaks
- Provide a centralized hub for study materials and notes
- AI-assisted study support via chat buddy

2) PAGE STRUCTURE & FLOW
================================================================================

ROUTES (defined in src/App.tsx):
- /                     -> TodaysLesson (default entry point)
- /dashboard           -> Index (CleanDashboard)
- /statistics          -> Statistics
- /weekly-summary      -> WeeklySummary
- /notes               -> NotesPage
- /achievements        -> AchievementsPage
- /calendar            -> CalendarPage
- /reminders           -> RemindersPage
- /resources           -> ResourcesPage
- /focus               -> FocusModePage
- /themes              -> ThemesPage
- /ai-tips             -> AITipsPage
- /theme-editor        -> ThemeEditorPage
- /*                   -> NotFound (404 catch-all)

ENTRY POINT:
- Root URL (/) loads TodaysLesson page
- Shows the next incomplete day from the 18-day schedule
- If all days complete, redirects to /dashboard

NAVIGATION LOGIC:
- FloatingNav component (fixed bottom-right)
- Click hamburger button to reveal 12-item grid menu
- Each menu item navigates to respective route
- Backdrop click closes menu

USER DECISION POINTS:
1. On TodaysLesson: Click lesson card to enter Focus Mode
2. In Focus Mode: Start/pause timer, write notes, mark complete
3. On Dashboard: Choose quick action or view subject progress
4. On Calendar: Click scheduled day to view lessons
5. On Notes: Search, view, rename, or export lessons

FLOW ORDER:
1. User lands on TodaysLesson
2. Selects a lesson to study
3. Enters Focus Mode with Pomodoro timer
4. Writes notes, marks complete
5. Confetti celebration on completion
6. Returns to lesson list or continues
7. When day complete, page reloads with next day

4) ANIMATIONS & MOTION
================================================================================

All animations use Framer Motion unless otherwise noted.

A) PAGE TRANSITIONS
   Trigger: Route change
   Duration: 500ms
   Easing: ease-out (default framer-motion)
   Transform: y: 20px -> 0, opacity: 0 -> 1
   Blocks input: No

B) CARD HOVER EFFECTS
   Trigger: mouseenter
   Duration: 300ms (Tailwind transition-all)
   Transform: scale(1.03), translateY(-4px)
   Purpose: Indicate interactivity

C) PROGRESS BAR ANIMATIONS
   Trigger: On mount
   Duration: 600-1000ms
   Easing: easeOut
   Transform: width 0% -> target%
   Purpose: Visual feedback of progress

D) PULSE ANIMATIONS (CSS)
   Trigger: Continuous
   Duration: 2000ms
   Animation: scale 1 -> 1.5, opacity 0.5 -> 0
   Purpose: Draw attention to buttons

E) FLOATING BUTTON PULSE
   Trigger: When menu closed
   Duration: 2000ms infinite
   Transform: scale 1 -> 1.5, opacity 0.5 -> 0
   Purpose: Indicate interactivity

F) CONFETTI EFFECT
   Trigger: Lesson completion, achievement unlock
   Duration: ~2000ms
   Library: canvas-confetti
   Particles: 100-150
   Spread: 70-80 degrees
   Purpose: Celebration/delight

G) MENU OPEN/CLOSE
   Trigger: Toggle button click
   Duration: ~300ms
   Easing: spring (stiffness: 200)
   Transform:
   - Open: opacity 0->1, scale 0.8->1
   - Close: opacity 1->0, scale 1->0.8
   Icon rotation: 0 -> 90 degrees
   Purpose: Smooth state transition

H) AMBIENT BACKGROUND ORBS
   Trigger: Continuous
   Duration: 12-18 seconds
   Easing: easeInOut
   Transform: x/y translation, scale 1-1.3
   Purpose: Visual polish

I) MODAL ANIMATIONS
   Trigger: Open/close
   Duration: ~300ms
   Transform:
   - Backdrop: opacity 0->1
   - Content: scale 0.8->1, y 50->0
   Purpose: Focus attention

J) TIMER DISPLAY PULSE
   Trigger: When timer running
   Duration: 2000ms infinite
   Transform: scale [1, 1.02, 1]
   Purpose: Indicate active timer

K) STREAK EMOJI PULSE
   Trigger: Continuous
   Duration: 2000ms infinite
   Transform: scale [1, 1.1, 1]
   Purpose: Highlight streak status

L) NEON TEXT PULSE (CSS)
   Trigger: Continuous
   Duration: 2000ms
   Effect: text-shadow intensity variation
   Purpose: Premium visual effect

6) INTERACTION LOGIC
================================================================================

A) LESSON COMPLETION FLOW:
   1. User clicks lesson card on TodaysLesson
   2. Focus mode activates (if lesson not complete)
   3. Timer starts automatically (25 min Pomodoro)
   4. User can pause/reset timer
   5. User writes notes in sidebar
   6. User clicks "Mark as Complete"
   7. Confetti fires, toast shows
   8. Progress saved to localStorage
   9. Returns to lesson list
   10. If all lessons done, page reloads to show next day

B) NAVIGATION INTERACTION:
   1. Click hamburger button (bottom-right)
   2. Menu grid appears with backdrop
   3. Click any nav item
   4. Navigates to route
   5. Menu closes automatically

C) AI CHAT INTERACTION:
   1. Click chat bubble button
   2. Chat window opens
   3. Type message or click quick action
   4. Press Enter to send
   5. Shows typing indicator
   6. AI response appears
   7. Continue conversation or close

D) CALENDAR INTERACTION:
   1. Navigate months with arrows
   2. Click any scheduled day (non-grayed)
   3. Detail panel expands below calendar
   4. Shows lessons with completion status
   5. Click X to close detail

E) NOTES SEARCH:
   1. Type in search input
   2. Real-time filtering of lesson cards
   3. Matches: name, topic, note content
   4. Click card to open detail modal

EDGE CASES:
- No lessons for today: Shows loading, then finds next incomplete day
- All lessons complete: Redirects to dashboard
- Empty search: Shows all lessons
- Timer at 0:00: Auto-stops, shows toast, confetti
- Network error in AI chat: Shows error message in chat

ERROR HANDLING:
- localStorage parse failures: console.error, use empty defaults
- OpenRouter API errors: Display error in chat window
- Invalid YouTube URL: Toast error message

DISABLED STATES:
- Completed lessons: Not clickable for focus mode, shows info toast
- Send button: Disabled when input empty or AI typing
- Navigation items: None disabled

- No server-side caching (no backend)

8) PERFORMANCE & TECH ASSUMPTIONS
================================================================================

RENDERING:
- React 18 with concurrent features available
- Client-side only (no SSR)
- Vite for bundling and HMR

LAZY LOADING:
- Not implemented for routes
- Could benefit from React.lazy() for code splitting

ANIMATION PERFORMANCE:
- Framer Motion uses GPU-accelerated transforms
- CSS animations use transform and opacity (GPU compositing)
- Backdrop-filter may cause performance issues on low-end devices
- Multiple animated background elements could impact mobile performance

BUNDLE CONSIDERATIONS:
- Heavy dependencies: framer-motion, recharts, openai
- Could benefit from tree-shaking unused Recharts components
- canvas-confetti is small (~3KB gzipped)

MEMORY:
- All data in memory via useState
- localStorage sync on every write
- No memory leaks expected (proper cleanup in useEffect)

EXPECTED ISSUES:
- Background animations may drop frames on mobile
- Blur effects expensive on low-end devices
- localStorage has 5MB limit

9) IMPROVEMENT OPPORTUNITIES (CRITICAL)
================================================================================

UX FRICTION POINTS:

1. Dark mode toggle not persisted - resets on refresh
   Impact: User frustration
   Fix: Add localStorage persistence

2. No loading states for AI chat API calls
   Impact: User uncertainty
   Fix: Already has typing indicator, but add error recovery

3. Focus mode timer auto-starts - unexpected behavior
   Impact: User may not be ready
   Fix: Add optional auto-start setting

4. YouTube input in Focus Mode not clearly discoverable
   Impact: Feature goes unused
   Fix: Add prominent button or panel

5. Calendar starts at November 2025 hardcoded
   Impact: May confuse users outside date range
   Fix: Dynamic start month based on schedule

6. No way to "uncomplete" a lesson
   Impact: Accidents can't be fixed
   Fix: Add toggle functionality

MISSED MICRO-INTERACTIONS:

1. Button press feedback - only scale, no color change
2. Form input focus animations missing
3. Navigation item active state not highlighted
4. No skeleton loading states
5. Toast notifications lack consistent icons

ACCESSIBILITY ISSUES:

1. No skip-to-content link
2. Floating nav not keyboard accessible
3. Color contrast may fail on glass backgrounds
4. No aria-labels on icon-only buttons
5. Timer not announced for screen readers
6. Confetti animation not reducible (motion preference)
7. Focus trap not implemented in modals

PERFORMANCE BOTTLENECKS:

1. 20 floating particles always rendering
2. Multiple background animations always running
3. No virtualization for lesson list
4. Recharts loads fully even if not viewed
5. OpenAI SDK loaded even if chat never used
6. No image optimization (none used, but consideration)

SCALABILITY LIMITATIONS:

1. localStorage 5MB limit for notes
2. No sync across devices
3. No user accounts
4. Schedule hardcoded, not configurable by user
5. Single language (English UI)
6. No data export/backup beyond markdown
